1//! Stabilizer ADC management interface
2//!
3//! # Design
4//!
5//! Stabilizer ADCs are connected to the MCU via a simplex, SPI-compatible interface. The ADCs
6//! require a setup conversion time after asserting the CSn (convert) signal to generate the ADC
7//! code from the sampled level. Once the setup time has elapsed, the ADC data is clocked out of
8//! MISO. The internal setup time is managed by the SPI peripheral via a CSn setup time parameter
9//! during SPI configuration, which allows offloading the management of the setup time to hardware.
10//!
11//! Because of the SPI-compatibility of the ADCs, a single SPI peripheral + DMA is used to automate
12//! the collection of multiple ADC samples without requiring processing by the CPU, which reduces
13//! overhead and provides the CPU with more time for processing-intensive tasks, like DSP.
14//!
15//! The automation of sample collection utilizes three DMA streams, the SPI peripheral, and two
16//! timer compare channel for each ADC. One timer comparison channel is configured to generate a
17//! comparison event every time the timer is equal to a specific value. Each comparison then
18//! generates a DMA transfer event to write into the SPI CR1 register to initiate the transfer.
19//! This allows the SPI interface to periodically read a single sample. The other timer comparison
20//! channel is configured to generate a comparison event slightly before the first (~10 timer
21//! cycles). This channel triggers a separate DMA stream to clear the EOT flag within the SPI
22//! peripheral. The EOT flag must be cleared after each transfer or the SPI peripheral will not
23//! properly complete the single conversion. Thus, by using two DMA streams and timer comparison
24//! channels, the SPI can regularly acquire ADC samples.
25//!
26//! In order to collect the acquired ADC samples into a RAM buffer, a final DMA transfer is
27//! configured to read from the SPI RX FIFO into RAM. The request for this transfer is connected to
28//! the SPI RX data signal, so the SPI peripheral will request to move data into RAM whenever it is
29//! available. When enough samples have been collected, a transfer-complete interrupt is generated
30//! and the ADC samples are available for processing.
31//!
32//! After a complete transfer of a batch of samples, the inactive buffer is available to the
33//! user for processing. The processing must complete before the DMA transfer of the next batch
34//! completes.
35//!
36//! ## Starting Data Collection
37//!
38//! Because the DMA data collection is automated via timer count comparisons and DMA transfers, the
39//! ADCs can be initialized and configured, but will not begin sampling the external ADCs until the
40//! sampling timer is enabled. As such, the sampling timer should be enabled after all
41//! initialization has completed and immediately before the embedded processing loop begins.
42//!
43//!
44//! ## Batch Sizing
45//!
46//! The ADCs collect a group of N samples, which is referred to as a batch. The size of the batch
47//! is configured by the user at compile-time to allow for a custom-tailored implementation. Larger
48//! batch sizes generally provide for lower overhead and more processing time per sample, but come
49//! at the expense of increased input -> output latency.
50//!
51//!
52//! # Note
53//!
54//! While there are two ADCs, only a single ADC is configured to generate transfer-complete
55//! interrupts. This is done because it is assumed that the ADCs will always be sampled
56//! simultaneously. If only a single ADC is used, it must always be ADC0, as ADC1 will not generate
57//! transfer-complete interrupts.
58//!
59//! There is a very small amount of latency between sampling of ADCs due to bus matrix priority. As
60//! such, one of the ADCs will be sampled marginally earlier before the other because the DMA
61//! requests are generated simultaneously. This can be avoided by providing a known offset to the
62//! sample DMA requests, which can be completed by setting e.g. ADC0's comparison to a counter
63//! value of 0 and ADC1's comparison to a counter value of 1.
64//!
65//! In this implementation, double buffer mode DMA transfers are used because the SPI RX FIFOs
66//! have finite depth, FIFO access is slower than AXISRAM access, and because the single
67//! buffer mode DMA disable/enable and buffer update sequence is slow.

86// The following data is written by the timer ADC sample trigger into the SPI CR1 to start the
87// transfer. Data in AXI SRAM is not initialized on boot, so the contents are random. This value is
88// initialized during setup.
89#[unsafe(link_section = ".axisram.buffers")]
90static SPI_START: GroundedCell<[u32; 1]> = GroundedCell::uninit();
91
92// The following data is written by the timer flag clear trigger into the SPI IFCR register to clear
93// the EOT flag. Data in AXI SRAM is not initialized on boot, so the contents are random. This
94// value is initialized during setup.
95#[unsafe(link_section = ".axisram.buffers")]
96static SPI_EOT_CLEAR: GroundedCell<[u32; 1]> = GroundedCell::uninit();

98// The following global buffers are used for the ADC sample DMA transfers. Two buffers are used for
99// each transfer in a ping-pong buffer configuration (one is being acquired while the other is being
100// processed). Note that the contents of AXI SRAM is uninitialized, so the buffer contents on
101// startup are undefined. The dimensions are `ADC_BUF[adc_index][ping_pong_index][sample_index]`.
102#[unsafe(link_section = ".axisram.buffers")]
103static ADC_BUF: GroundedArrayCell<[SampleBuffer; 2], 2> =
104    GroundedArrayCell::uninit();

106macro_rules! adc_input {
107    ($name:ident, $index:literal, $trigger_stream:ident, $data_stream:ident, $clear_stream:ident,
108     $spi:ident, $trigger_channel:ident, $dma_req:ident, $clear_channel:ident, $dma_clear_req:ident) => {
109
110        paste::paste! {
111
112            /// $spi-CR is used as a type for indicating a DMA transfer into the SPI control
113            /// register whenever the tim2 update dma request occurs.
114            struct [< $spi CR >] {
115                _channel: timers::tim2::$trigger_channel,
116            }
117            impl [< $spi CR >] {
118                pub fn new(_channel: timers::tim2::$trigger_channel) -> Self {
119                    Self { _channel }
120                }
121            }
122
123            // Note(unsafe): This structure is only safe to instantiate once. The DMA request is
124            // hard-coded and may only be used if ownership of the timer2 $trigger_channel compare
125            // channel is assured, which is ensured by maintaining ownership of the channel.
126            unsafe impl TargetAddress<MemoryToPeripheral> for [< $spi CR >] {
127
128                type MemSize = u32;
129
130                /// SPI DMA requests are generated whenever TIM2 CHx ($dma_req) comparison occurs.
131                const REQUEST_LINE: Option<u8> = Some(DMAReq::$dma_req as u8);
132
133                /// Whenever the DMA request occurs, it should write into SPI's CR1 to start the
134                /// transfer.
135                fn address(&self) -> usize {
136                    // Note(unsafe): It is assumed that SPI is owned by another DMA transfer. This
137                    // is only safe because we are writing to a configuration register.
138                    let regs = unsafe { &*hal::stm32::$spi::ptr() };
139                    &regs.cr1 as *const _ as usize
140                }
141            }
142
143            /// $spi-IFCR is used as a type for indicating a DMA transfer into the SPI flag clear
144            /// register whenever the tim3 compare dma request occurs. The flag must be cleared
145            /// before the transfer starts.
146            struct [< $spi IFCR >] {
147                _channel: timers::tim3::$clear_channel,
148            }
149
150            impl [< $spi IFCR >] {
151                pub fn new(_channel: timers::tim3::$clear_channel) -> Self {
152                    Self { _channel }
153                }
154            }
155
156            // Note(unsafe): This structure is only safe to instantiate once. The DMA request is
157            // hard-coded and may only be used if ownership of the timer3 $clear_channel compare
158            // channel is assured, which is ensured by maintaining ownership of the channel.
159            unsafe impl TargetAddress<MemoryToPeripheral> for [< $spi IFCR >] {
160                type MemSize = u32;
161
162                /// SPI DMA requests are generated whenever TIM3 CHx ($dma_clear_req) comparison
163                /// occurs.
164                const REQUEST_LINE: Option<u8> = Some(DMAReq::$dma_clear_req as u8);
165
166                /// Whenever the DMA request occurs, it should write into SPI's IFCR to clear the
167                /// EOT flag to allow the next transmission.
168                fn address(&self) -> usize {
169                    // Note(unsafe): It is assumed that SPI is owned by another DMA transfer and
170                    // this DMA is only used for writing to the configuration registers.
171                    let regs = unsafe { &*hal::stm32::$spi::ptr() };
172                    &regs.ifcr as *const _ as usize
173                }
174            }
175
176            /// Represents data associated with ADC.
177            pub struct $name {
178                transfer: Transfer<
179                    hal::dma::dma::$data_stream<hal::stm32::DMA1>,
180                    hal::spi::Spi<hal::stm32::$spi, hal::spi::Disabled, u16>,
181                    PeripheralToMemory,
182                    &'static mut [u16],
183                    hal::dma::DBTransfer,
184                >,
185                trigger_transfer: Transfer<
186                    hal::dma::dma::$trigger_stream<hal::stm32::DMA1>,
187                    [< $spi CR >],
188                    MemoryToPeripheral,
189                    &'static mut [u32; 1],
190                    hal::dma::DBTransfer,
191                >,
192                clear_transfer: Transfer<
193                    hal::dma::dma::$clear_stream<hal::stm32::DMA1>,
194                    [< $spi IFCR >],
195                    MemoryToPeripheral,
196                    &'static mut [u32; 1],
197                    hal::dma::DBTransfer,
198                >,
199            }
200
201            impl $name {
202                /// Construct the ADC input channel.
203                ///
204                /// # Args
205                /// * `spi` - The SPI interface used to communicate with the ADC.
206                /// * `trigger_stream` - The DMA stream used to trigger each ADC transfer by
207                ///    writing a word into the SPI TX FIFO.
208                /// * `data_stream` - The DMA stream used to read samples received over SPI into a data buffer.
209                /// * `clear_stream` - The DMA stream used to clear the EOT flag in the SPI peripheral.
210                /// * `trigger_channel` - The ADC sampling timer output compare channel for read triggers.
211                /// * `clear_channel` - The shadow sampling timer output compare channel used for
212                ///   clearing the SPI EOT flag.
213                pub fn new(
214                    spi: hal::spi::Spi<hal::stm32::$spi, hal::spi::Enabled, u16>,
215                    trigger_stream: hal::dma::dma::$trigger_stream<
216                        hal::stm32::DMA1,
217                    >,
218                    data_stream: hal::dma::dma::$data_stream<hal::stm32::DMA1>,
219                    clear_stream: hal::dma::dma::$clear_stream<hal::stm32::DMA1>,
220                    trigger_channel: timers::tim2::$trigger_channel,
221                    clear_channel: timers::tim3::$clear_channel,
222                    batch_size: usize,
223                ) -> Self {
224                    // The flag clear DMA transfer always clears the EOT flag in the SPI
225                    // peripheral. It has the highest priority to ensure it is completed before the
226                    // transfer trigger.
227                    let clear_config = DmaConfig::default()
228                        .priority(Priority::VeryHigh)
229                        .circular_buffer(true);
230
231                    // Note(unsafe): Because this is a Memory->Peripheral transfer, this data is
232                    // never actually modified. It technically only needs to be immutably
233                    // borrowed, but the current HAL API only supports mutable borrows.
234                    let spi_eot_clear = unsafe {
235                        let ptr = SPI_EOT_CLEAR.get();
236                        ptr.write([1 << 3]);
237                        &mut *ptr
238                    };
239
240                    // Generate DMA events when the timer hits zero (roll-over). This must be before
241                    // the trigger channel DMA occurs, as if the trigger occurs first, the
242                    // transmission will not occur.
243                    clear_channel.listen_dma();
244                    clear_channel.to_output_compare(0);
245
246                    let clear_transfer: Transfer<
247                        _,
248                        _,
249                        MemoryToPeripheral,
250                        _,
251                        _,
252                    > = Transfer::init(
253                        clear_stream,
254                        [< $spi IFCR >]::new(clear_channel),
255                        spi_eot_clear,
256                        None,
257                        clear_config,
258                    );
259
260                    // Generate DMA events when an output compare of the timer hits the specified
261                    // value.
262                    trigger_channel.listen_dma();
263                    trigger_channel.to_output_compare(2 + $index);
264
265                    // The trigger stream constantly writes to the SPI CR1 using a static word
266                    // (which is a static value to enable the SPI transfer).  Thus, neither the
267                    // memory or peripheral address ever change. This is run in circular mode to be
268                    // completed at every DMA request.
269                    let trigger_config = DmaConfig::default()
270                        .priority(Priority::High)
271                        .circular_buffer(true);
272
273                    // Note(unsafe): This word is initialized once per ADC initialization to verify
274                    // it is initialized properly.
275                    // Note(unsafe): Because this is a Memory->Peripheral transfer, this data is never
276                    // actually modified. It technically only needs to be immutably borrowed, but the
277                    // current HAL API only supports mutable borrows.
278                    // Write a binary code into the SPI control register to initiate a transfer.
279                    let spi_start = unsafe {
280                        let ptr = SPI_START.get();
281                        ptr.write([0x201]);
282                        &mut *ptr
283                    };
284
285                    // Construct the trigger stream to write from memory to the peripheral.
286                    let trigger_transfer: Transfer<
287                        _,
288                        _,
289                        MemoryToPeripheral,
290                        _,
291                        _,
292                    > = Transfer::init(
293                        trigger_stream,
294                        [< $spi CR >]::new(trigger_channel),
295                        spi_start,
296                        None,
297                        trigger_config,
298                    );
299
300                    // The data stream constantly reads from the SPI RX FIFO into a RAM buffer. The peripheral
301                    // stalls reads of the SPI RX FIFO until data is available, so the DMA transfer completes
302                    // after the requested number of samples have been collected. Note that only ADC1's (sic!)
303                    // data stream is used to trigger a transfer completion interrupt.
304                    let data_config = DmaConfig::default()
305                        .memory_increment(true)
306                        .double_buffer(true)
307                        .transfer_complete_interrupt($index == 1)
308                        .priority(Priority::VeryHigh);
309
310                    // A SPI peripheral error interrupt is used to determine if the RX FIFO
311                    // overflows. This indicates that samples were dropped due to excessive
312                    // processing time in the main application (e.g. a second DMA transfer completes
313                    // before the first was done with processing). This is used as a flow control
314                    // indicator to guarantee that no ADC samples are lost.
315                    let mut spi = spi.disable();
316                    spi.listen(hal::spi::Event::Error);
317
318                    let adc_bufs = unsafe {
319                        ADC_BUF.initialize_all_with(|| Default::default());
320                        ADC_BUF.get_element_mut_unchecked($index).split_at_mut(1)
321                    };
322
323                    // The data transfer is always a transfer of data from the peripheral to a RAM
324                    // buffer.
325                    let data_transfer: Transfer<_, _, PeripheralToMemory, _, _> =
326                        Transfer::init(
327                            data_stream,
328                            spi,
329                            // Note(unsafe): The ADC_BUF[$index] is "owned" by this peripheral.
330                            // It shall not be used anywhere else in the module.
331                            &mut adc_bufs.0[0][..batch_size],
332                            Some(&mut adc_bufs.1[0][..batch_size]),
333                            data_config,
334                        );
335
336                    Self {
337                        transfer: data_transfer,
338                        trigger_transfer,
339                        clear_transfer,
340                    }
341                }
342
343                /// Enable the ADC DMA transfer sequence.
344                pub fn start(&mut self) {
345                    self.transfer.start(|spi| {
346                        spi.enable_dma_rx();
347
348                        spi.inner().cr2.modify(|_, w| w.tsize().bits(1));
349                        spi.inner().cr1.modify(|_, w| w.spe().set_bit());
350                    });
351
352                    self.clear_transfer.start(|_| {});
353                    self.trigger_transfer.start(|_| {});
354
355                }
356
357                /// Wait for the transfer of the currently active buffer to complete,
358                /// then call a function on the now inactive buffer and acknowledge the
359                /// transfer complete flag.
360                ///
361                /// NOTE(unsafe): Memory safety and access ordering is not guaranteed
362                /// (see the HAL DMA docs).
363                pub fn with_buffer<F, R>(&mut self, f: F) -> Result<R, DMAError>
364                where
365                    F: FnOnce(&mut &'static mut [u16]) -> R,
366                {
367                    unsafe { self.transfer.next_dbm_transfer_with(|buf, _current| {
368                        f(buf)
369                    })}
370                }
371            }
372
373            // This is not actually a Mutex. It only re-uses the semantics and macros of mutex-trait
374            // to reduce rightward drift when jointly calling `with_buffer(f)` on multiple DAC/ADCs.
375            impl Mutex for $name {
376                type T = &'static mut [u16];
377                fn lock<R>(&mut self, f: impl FnOnce(&mut Self::T) -> R) -> R {
378                    self.with_buffer(f).unwrap()
379                }
380            }
381        }
382    };
383}
